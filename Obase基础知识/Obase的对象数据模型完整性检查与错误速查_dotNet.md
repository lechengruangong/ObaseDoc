完整性检查是对Obase对象数据模型的校验,分为对象类型通用检查,实体型检查和关联型检查三种.

顾名思义,实体型检查和关联型检查就是对实体型和关联型分别生效的检查,而对象类型通用检查是同时对实体型和关联生效的检查,如果完整性检查没有通过,会抛出特定类型的异常IntegrityCheckFailException.

IntegrityCheckFailException包含一个Dictionary<string, List<string>>类型的错误消息字典ErrorMessageDictionary,在此字典中可以查看到当前检查失败的消息.

ErrorMessageDictionary的键是类型名称,值是错误消息列表,类型名称包含以下三种:
1. 实体型,就是类型的名称.
2. 显式关联型,就是类型的名称.
3. 隐式关联型,是一个特殊的内部名称,分为三个部分:
   1. 第一部分固定为ImplicitAssociation表示这是一个隐式关联型.
   2. 第二部分为参与此关联各个关联端的类型名称,以下划线作为分隔符.
   3. 第三部分为<>数字组成的定义顺序.

以下是目前已有的完整性检查中错误的速查表:

| 错误信息                                   | 需要检查的配置或修改方法                                 |
|----------------------------------------|----------------------------------------------|
| XXX未配置有效的构造函数.                         | 类内没有合适的无参构造函数,需要手动后调用HasConstructor方法指定构造器.  |
| XXX未配置映射表.                             | 检查ToTable方法配置的映射表是不是空.                       |
| XXX没有配置具体类型判别标志.                       | 检查是否使用HasConcreteTypeSign方法配置了具体类型判别标志.      |
| XXX映射表与父类不一致                           | 检查继承的父类配置ToTable方法配置的映射表和子类是否一致.             |
| XXX具体类型判别标志配置中自己和自己的子类中存在重复的具体类型判别标志值. | 检查使用HasConcreteTypeSign方法配置的值是否有重复.          |
| XXX构造器参数个数与父类参数个数不一致.                  | 检查子类内是否存在与父类的构造器使用的构造函数参数一致的构造函数并且是否被配置为构造器. |
| XXX构造器参数第X个参数类型与父类参数类型不一致.             | 检查子类内是否存在与父类的构造器使用的构造函数参数一致的构造函数并且是否被配置为构造器. |
| XXX没有设值器.                              | 检查类的属性是否有Set方法或者自己配置的设值器是否是空.                |
| XXX没有取值器.                              | 检查类的属性是否有Get方法或者自己配置的取值器是否是空.                |
| XXX启用了延迟加载,但没有将其声明为virtual的.           | 检查启用了延迟加载的属性,必须是virtual的.                    |
| XXX的键属性未设置.                            | 检查是否设置了主键.                                   |
| XXX的键属性是联合主键,不能是自增的.                   | 检查是否为设置了多个主键的类型设置了主键自增.                      |
| XXX是自增的但不是short,int,long类型.            | 检查是不是将不能自增的主键类型设置了主键自增.                      |
| XXX的左端和右端不能相同.                         | 检查是不是将同一个关联引用配置了多次.                          |
| XXX关联表不能是自身的映射.                        | 检查是不是将一对多的关联引用的映射表配置错了.                      |
| XXX内应只有关联端.                            | 检查是不是手动指定了一个显式关联类的Visible为false.             |
| XXX的关联端没有映射.                           | 检查是不是没配置关联端映射.                               |
| XXX关联端映射性无法在端类型中找到.                    | 检查关联端的映射第一个参数是不是写错了.                         |
| XXX关联端的标识属性应有且只1个映射.                   | 检查关联端的映射第一个参数是不是和关联端实体型主键一致.                 |

在大多数情况下,完整性检查都是帮助我们进行模型校验不可或缺的一步,但如果你确定自己的配置没有任何问题但是和完整性检查相违背,可以在配置提供器CreateModel(ModelBuilder modelBuilder)的参数modelBuilder上调用HasIntegrityCheck(false)来关闭完整性检查.
