在使用Obase的过程中,虽然有完整性检查等手段来辅助使用者配置模型,并且开发者也在尽力避免意外的异常,但有些时候还是会出现抛出无具体异常信息的异常. 此时为了确定具体的问题就需要对软件包进行调试,以下就介绍如何对Obase进行调试.

从6.4版本开始,Obase已经在GitHub上开源并且打包了文档和源代码,可以通过一些设置来启用对软件包的调试,这里以IDEA为例来讲解具体的步骤:
1. 引用6.4+版本的Obase框架.
2. 在MAVEN包管理界面,找到依赖项中io.obase的包.
3. 找到需要进行调试的包,右键点击选择下载源代码和文档.

在下载了源代码和文档后后可以从以下入口进入调试.

- 保存时出现异常,可以在对象上下文的saveChanges方法上打断点后使用单步模式进入方法内部进行调试.
- 查询时出现异常,可以在对象上集合调用的ObaseStream接口定义的方法上打断点,单步模式进入方法内部进行调试.

更多的代码内容可以查看Obase的开源项目.
