# Obase框架相关文档
本项目为Obase框架的相关文档,Obase是一款存储抽象层框架中间件,分别在dotNet和Java两种语言平台发布软件包,旨在为开发者提供一个统一的存储抽象层,使得开发者可以更专注于业务逻辑的实现,而不必过多关注底层存储的细节.

Obase框架是开源的并且使用GitHub Action发布软件包,以下为开源仓库地址:
- [Obase框架在dotNet平台的实现](https://github.com/lechengruangong/Obase4DotNet)
- [Obase框架在Java平台的实现](https://github.com/lechengruangong/Obase4Java)

## 目录结构
本文档相关的目录结构如下:
- Obase入门:此目录下是Obase的入门知识,为初次使用Obase的用户提供入门引导.
- Obase基础知识:此目录下为Obase基础知识,主要包含Obase的相关基础知识和讲解如何进行Obase配置.
- Obase进阶使用:此目录下为Obase的进阶使用文档,包含解决特定问题而开发的功能文档,特殊的配置示例和Obase可以二次开发的扩展点.
  
由于不同的语言平台中大部分Api的命名规则,参数等不一致,故大部分文档是区分具体的语言的,阅读时需根据自己所使用的语言进行选择,结尾是_dotNet的为dotNet平台的文档,结尾是_java的为java平台文档.

也有部分文档的内容无关具体的语言平台,这部分文档则没有后缀,统一阅读即可.

## 新手阅读推荐
如果是首次接触Obase,推荐按照以下顺序阅读这些文档:
1. 阅读Obase入门中的文档,即按照顺序依次阅读相应语言的快速开始,最佳实践,深入理解,代码优先这四篇文档.
2. 阅读Obase基础知识中相应语言的Obase的总体配置思路和Obase对象数据模型配置示例这两篇文档来了解如何配置Obase对象数据模型.
3. 阅读Obase基础知识中相应语言的Obase的配置提供器有哪些可以重写的方法和属性这篇文档来了解如何定义上下文和上下文配置提供器,阅读Obase基础知识中相应语言的Obase上下文管理来了解如何使用Obase的对象上下文.. 
4. 阅读Obase基础知识中相应语言的Obase支持的关系运算来了解Obase支持哪些查询操作.

阅读完这些内容后,即可使用Obase实现通用场景下的对象查询,存储,变更追踪等功能.

## 解决特定问题
除了之前介绍的常规使用之外,Obase框架还提供了很多其他功能来解决存储过程中的特定场景的问题,如并发处理,逻辑删除,复杂序列化属性等问题.

同时还有一些特殊的配置,如自关联,聚合关联端,关联端默认附加以及调试相关和对Obase进行二次开发相关的内容,这些内容都放置于Obase进阶使用目录中,如果遇到某些特定的场景,可以先阅读Obase进阶使用目录中的README来了解如何来解决特定的问题.
